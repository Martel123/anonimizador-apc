PASO 1: Deshabilitar 2FA “para siempre”
A) Backend:
- Agrega flag global: app.config["ENABLE_2FA"] = False
- Rodea toda lógica de 2FA con if app.config.get("ENABLE_2FA", False)
- Si hay verificación/post-login que redirige a /verificar_2fa, debe quedar desactivado.
B) Ruta:
- Mantén /verificar_2fa pero reemplázala para que haga return redirect(url_for("login")) (o "/login").
C) DB:
- Agrega un script utilitario (o comando copy-pasteable) para apagar 2FA en todos los usuarios si existen columnas como: two_factor_enabled, two_factor_secret, twofa_*.
- Si no existen esas columnas, no falles.

PASO 2: Política de contraseña fuerte (registro y reset)
Implementa función:
- mínimo 10 caracteres
- al menos 1 mayúscula, 1 minúscula, 1 número, 1 símbolo
- NO permitir contraseñas comunes básicas ("password", "123456", "qwerty", "admin", etc.)
Aplica validación:
- En registro (y reset si existe)
- Mensaje de error claro y humano

PASO 3: Máximo 5 intentos de login (anti brute force)
Sin librerías externas.
- Crea tabla LoginAttempt (SQLAlchemy) o usa una tabla existente si hay algo similar.
Campos recomendados:
  id, email (nullable), ip, user_agent (nullable), success (bool),
  created_at (datetime), reason (str nullable)
- Regla:
  Si en los últimos 15 minutos hay >= 5 intentos fallidos para (email+ip) o para ip sola -> bloquear login temporalmente (HTTP 429 o flash message).
- Cuando bloqueo: registrar reason="rate_limited".

PASO 4: Logs de login
- Agrega tabla LoginEvent o reutiliza LoginAttempt para también registrar exitosos.
- En cada login exitoso y fallido guarda: email, ip, user_agent, timestamp, success, reason.
- Asegura que el log no guarde contraseñas ni tokens.

PASO 5: Alertas por “IP rara”
Sin servicios externos.
Implementación simple:
- En login exitoso, compara la IP actual con las últimas IPs exitosas del usuario (por ejemplo últimas 5).
- Si IP nueva y usuario ya tenía historial, marca evento con flag "ip_unusual" y muestra un banner en UI después del login:
  “Nuevo inicio de sesión desde una IP no habitual: X. Si no fuiste tú, cambia tu contraseña.”
- (Opcional) Si existe mailer en el repo, también envía correo; si no, no inventes.
- También agrega un endpoint /account/security (protegido) donde el usuario vea: últimos 10 logins + IPs.
