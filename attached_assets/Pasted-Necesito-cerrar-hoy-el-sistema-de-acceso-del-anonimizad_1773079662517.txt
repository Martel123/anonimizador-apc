PROBLEMA 2 — CONFIRMAR QUE EL EMAIL SEA REAL
========================================

Quiero implementar verificación de correo del anonimizador para confirmar que el email del usuario es real.

Objetivo:
Cuando un usuario se registre, debe recibir un correo de verificación. Hasta verificar su email, su cuenta no debe considerarse plenamente confirmada.

Qué quiero exactamente:

1. Revisar el modelo User.
   Si no existe un campo como:
   - email_verified
   créalo con valor por defecto False.

2. En el registro:
   - cuando se cree el usuario, dejar email_verified = False
   - generar un token seguro de verificación
   - enviar correo con enlace de verificación usando la infraestructura actual de email/Resend

3. Crear ruta de verificación, por ejemplo:
   /verify-email/<token>
   o la ruta equivalente coherente con el proyecto

4. Cuando el usuario haga clic:
   - validar token
   - verificar expiración
   - marcar email_verified = True
   - invalidar token si corresponde
   - redirigir a login o dashboard con mensaje de éxito

5. Seguridad:
   - no revelar en pantalla si el correo existe o no cuando corresponda
   - token con expiración razonable (24h máximo, ideal 1h-24h)
   - reutilizar la infraestructura de tokens si ya existe una clase tipo ActivationToken o similar
   - no guardar contraseñas ni tokens inseguros

6. UI:
   - si el usuario no verificó su correo, mostrar un aviso claro después del login o en el panel
   - agregar opción "Reenviar correo de verificación"
   - no romper usuarios existentes

7. Restricción funcional:
   Quiero que, hasta que el correo no esté verificado, el usuario NO pueda ejecutar acciones críticas, idealmente:
   - anonimizar documentos
   - descargar resultados
   - comprar créditos
   Elige el punto de control más simple y seguro del proyecto y aplícalo ahí.

8. Compatibilidad:
   - usuarios ya existentes no deben romperse
   - si ya hay usuarios antiguos, define una estrategia segura:
     - o asumir email_verified = True para legacy users
     - o manejar migración clara
   Elige la opción más estable para no romper la demo de hoy y explícala.
